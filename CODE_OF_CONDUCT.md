# Code of Conduct

SecureStore contributors are expected to keep discussion respectful, security-conscious, and constructive.

## Expected Behavior

- Be kind and specific in feedback.
- Keep security reports responsible and private until triaged.
- Avoid sharing real secrets, vault data, or personal credentials in issues or pull requests.

## Unacceptable Behavior

- Harassment, discrimination, or personal attacks.
- Publishing exploit details before maintainers can respond.
- Posting sensitive user data or secrets.

Maintainers may remove content or reject contributions that violate this code.
