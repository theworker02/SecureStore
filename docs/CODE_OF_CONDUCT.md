# Code of Conduct

## Our Commitment

SecureStore contributors agree to keep this project welcoming, respectful, and constructive for everyone.

## Expected Behavior

- Be respectful in reviews, issues, and discussions.
- Focus feedback on the work, not the person.
- Share context, reproduction steps, and evidence when reporting bugs.
- Assume good intent and collaborate toward a fix.

## Unacceptable Behavior

- Harassment, intimidation, or discriminatory language.
- Personal attacks or repeated hostile behavior.
- Sharing private, sensitive, or security-relevant information irresponsibly.
- Publishing exploit details before maintainers have time to assess them.

## Enforcement

Project maintainers may edit, remove, or reject contributions and discussions that violate this code of conduct.

## Reporting

For conduct concerns, open a private maintainer contact request instead of posting publicly.
